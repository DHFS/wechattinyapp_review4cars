# 数据库权限配置指南

## ⚠️ 重要：必须正确配置数据库权限

### reviews 集合权限设置

在微信开发者工具 -> 云开发 -> 数据库 -> reviews -> 权限设置：

**推荐配置（安全）：**
```json
{
  "read": "doc._openid == auth.openid",
  "write": "doc._openid == auth.openid"
}
```

这个配置确保：
- 用户只能读取自己创建的评价
- 用户只能修改/删除自己创建的评价
- 无法读取其他人的评价数据

### 如果需要在详情页显示所有评价

详情页需要显示某车型的所有评价，这需要特殊处理：

**方案1：使用云函数查询（推荐）**
- 创建云函数 `getCarReviews` 
- 云函数有管理员权限，可以查询所有数据
- 小程序端调用云函数获取某车型的所有评价

**方案2：调整权限为仅创建者可写，所有人可读**
```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```
⚠️ 注意：这样所有人都能看到所有评价内容，仅限制修改权限

### cars 集合权限设置

```json
{
  "read": true,
  "write": "auth.openid != null"
}
```
- 所有人可读（排行榜需要）
- 登录用户可写（添加车型）

## 调试建议

1. 打开微信开发者工具控制台
2. 查看首页加载时的日志：
   - "获取到 X 个头像" - 应该显示3个（如果有3条评价）
   - "原始评价数据" - 检查评价数据是否正确
3. 查看"我的点评"页面日志：
   - "加载我的评价, openid: xxx" - 确认 openid 正确获取
   - "查询到的评价数据" - 确认只返回当前用户的数据

## 常见问题

### Q: 头像只显示一个？
A: 检查数据库中该车型是否有3条不同的评价，且每条评价都有 user_avatar 字段

### Q: "我的点评"显示他人数据？
A: 立即检查数据库权限配置，确保 reviews 集合 read 权限为 `doc._openid == auth.openid`

### Q: 详情页看不到任何评价？
A: 如果权限设置为仅创建者可读，详情页需要改为云函数查询方式
