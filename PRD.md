# 车评侦探 - 产品需求文档（PRD）

> **版本**: v1.4.2  
> **更新日期**: 2026-04-15  
> **产品类型**: 微信小程序（汽车评价类）  
> **目标用户**: 汽车爱好者、潜在购车用户、车主

---

## 1. 产品概述

### 1.1 产品定位

**车评侦探**是一款基于真实用户评价的**车型评分小程序**，采用 **Editorial Industrialism**（编辑工业主义）设计风格，让用户可以对不同车型进行五维度专业评分，查看车型排行榜，管理个人评价。

### 1.2 核心价值主张

| 维度 | 价值点 |
|:---|:---|
| **对用户** | 获取真实车主评价，辅助购车决策 |
| **对车主** | 分享用车体验，帮助其他车友 |
| **对产品** | 建立基于五维评分的车型口碑数据库 |

### 1.3 竞品对标

- **懂车帝/汽车之家**: 专业媒体评测为主，缺乏车主五维量化评分
- **小红书**: 零散口碑，缺乏结构化评分体系
- **本产品差异化**: 专注五维量化评分 + 雷达图可视化 + UGC评价聚合

## 3. 功能模块详解

### 3.1 首页 - 车型排行榜

**页面路径**: `pages/index/index`

#### 功能描述
展示所有车型的综合评分排行榜，采用大图卡片设计，并基于贝叶斯平滑后的 `rank_score` 排序。

#### 核心功能点

| 功能点 | 详细说明 |
|:---|:---|
| **大图卡片展示** | 车型图片占满卡片上半部分（520rpx高度），圆角融合设计 |
| **排名展示** | 大号斜体排名数字作为背景装饰（01, 02, 03...） |
| **动力类型标签** | 胶囊形毛玻璃标签，四种颜色区分动力形式 |
| **评分展示** | 右下角大数字评分（100rpx字号），#FF6B35强调色 |
| **评价人数展示** | 卡片同时展示评价人数，辅助解释分数可信度 |
| **榜单状态分层** | 区分 `official / pending / observation`，正式榜与观察期车型分层展示 |
| **评价用户头像堆叠** | 最多展示3个最近评价用户头像 |
| **下拉刷新** | 支持下拉刷新获取最新数据 |
| **加载更多** | 分页加载，点击加载更多 |

#### 交互说明
- 点击卡片 → 跳转车型详情页
- 下拉 → 刷新列表
- 上拉底部 → 加载更多

### 3.2 车型详情页

**页面路径**: `pages/detail/detail`

#### 功能描述
展示单一车型的详细信息、五维雷达图、用户评价列表，支持用户提交评价。

#### 核心功能点

| 模块 | 功能说明 |
|:---|:---|
| **车型封面图** | 大图展示，已登录用户可长按提交纠错/新图，图片进入审核队列 |
| **基本信息** | 品牌+车型名称、动力类型标签、年款、价格区间 |
| **综合得分** | 右上角大数字展示 |
| **样本量说明** | 展示评价人数与榜单状态，避免小样本误解 |
| **五维雷达图** | Canvas绘制，展示全网平均分在各维度表现 |
| **五维评分标签** | 紧凑布局展示各维度分数及权重占比 |
| **评价入口** | 已登录未评价用户显示"撰写评价"按钮；已评价显示"修改评价" |
| **评价列表** | 展示所有用户评价，含头像、昵称、时间、分数、内容、五维分数 |
| **评价操作** | 用户可编辑/删除自己的评价 |
| **评论展开/收起** | 长评论支持展开收起 |
| **完善资料弹窗** | 未完善资料的用户需先设置头像昵称才能评价 |

#### 五维评分体系

| 维度 | 权重 | 说明 |
|:---|:---:|:---|
| 动力三电 | 30% | 电池、电机、电控性能 |
| 操控底盘 | 20% | 悬挂、转向、驾驶感受 |
| 空间内饰 | 20% | 乘坐空间、内饰质感 |
| 辅驾安全 | 20% | 辅助驾驶、安全配置 |
| 其他体验 | 10% | 智能座舱、售后服务等 |

**综合分计算公式**: `动力×0.3 + 操控×0.2 + 空间×0.2 + 辅驾×0.2 + 其他×0.1`

### 3.3 添加车型页

**页面路径**: `pages/addCar/addCar`

#### 功能描述
用户填写新车型基础信息，支持品牌/车型联想、智能纠错。最终车型会与首评分、首条评论一起打包提交审核。

#### 表单字段

| 字段 | 类型 | 必填 | 说明 |
|:---|:---:|:---:|:---|
| 品牌厂家 | 文本 | ✅ | 支持本地联想，含别名识别 |
| 车型名称 | 文本 | ✅ | 基于已选品牌联想 |
| 年款 | 文本 | ✅ | 如"2024款" |
| 动力形式 | 单选 | ✅ | 纯电/增程/插混/燃油 |
| 售价区间 | 文本 | ✅ | 如"23.19-33.19万" |
| 官方车型图片 | 图片 | ❌ | 可作为首图一起提交，最终由管理员审核决定是否展示 |

#### 智能联想功能

| 功能 | 说明 |
|:---|:---|
| **品牌联想** | 输入时实时显示匹配的品牌列表（支持别名） |
| **车型联想** | 选择品牌后，输入车型时显示该品牌车型列表 |
| **自动填充** | 选择联想结果后，自动带出动力形式和售价区间 |
| **智能纠错** | 提交前调用云函数进行品牌和车型纠错匹配 |

#### 技术实现
- 本地车型词典 `carDictionary.js` 支持快速联想
- 云函数 `suggestCarModels` 提供智能纠错和补全
- 词典包含品牌/车型别名映射（如"BYD"→"比亚迪"）
- 本页生成的是车型草稿，最终由评价页统一提交审核

### 3.4 撰写评价页

**页面路径**: `pages/writeReview/writeReview`

#### 功能描述
两步式评价流程：第一步五维评分，第二步撰写文字评价。

#### 第一步：五维评分

| 功能 | 说明 |
|:---|:---|
| **实时综合分计算** | 根据权重实时计算并显示综合评分 |
| **滑块评分** | 每个维度0-100分滑块，橙色主题 |
| **快捷评分按钮** | 50/60/70/80/90分一键设置 |
| **下一步按钮** | 完成评分后进入撰写步骤 |

#### 第二步：撰写评价 / 提交审核

| 功能 | 说明 |
|:---|:---|
| **评分摘要展示** | 顶部固定展示综合分和五维分数 |
| **全屏文本输入** | 大文本框，支持自动高度 |
| **字数统计** | 实时显示已输入字数（最多2000字） |
| **返回修改** | 可返回上一步修改评分 |
| **发布/保存** | 已存在车型评价显示"发布/保存"；新车型首评显示"提交审核" |

#### 新车型整包提交流程

当用户从“添加车型”进入本页时，最终提交会一次性创建：
- `cars.status = pending` 的新车型
- `reviews.status = pending` 的首评
- 可选首图

管理员审核通过后，车型、评分、评论才会对全站公开。

### 3.6 审核中心

**页面路径**: `pages/adminCars/adminCars`

#### 功能描述
管理员在小程序内统一处理两类待审核任务，不再依赖云控制台手动改库。

#### 审核任务类型

| 类型 | 展示内容 | 可执行动作 |
|:---|:---|:---|
| **新车型整包审核** | 车型基础信息、首图、五维分、首条评论 | 通过 / 拒绝 |
| **车型图片审核** | 车型信息、当前正式图、用户新提交图片 | 通过图片 / 拒绝图片 |

#### 说明
- 图片审核支持多张待审核图片排队处理
- 审核通过的新车型会立即写回首评分数，避免首页卡片出现 0 分
- 图片审核通过后才会替换正式图；拒绝后会清理待审核文件

## 4. 设计规范

### 4.1 设计风格

**Editorial Industrialism（编辑工业主义）**

将小程序从普通的"应用感"转变为**高端汽车杂志/技术档案**的视觉体验，结合工业机械的粗犷感与高端汽车杂志的精致排版。

### 4.2 核心设计原则

| 原则 | 说明 |
|:---|:---|
| **无分割线设计** | 仅用色块和留白区分区域 |
| **大图优先** | 车型封面图作为视觉焦点 |
| **深色沉浸** | Carbon & Combustion 主题色调 |
| **编辑式排版** | 杂志级别的文字层级 |

### 4.3 色彩系统

#### 基础色板

| Token | 色值 | 用途 |
|:---|:---|:---|
| `surface` | #131313 | 页面背景 |
| `surface-container-low` | #1c1b1b | 卡片背景 |
| `on-surface` | #e5e2e1 | 主文字 |
| `primary` | #ff6b35 | 强调色（评分、按钮） |
| `text-secondary` | #B0B0B0 | 次要文字 |
| `text-tertiary` | #707070 | 辅助文字 |

#### 动力类型标签色

| 类型 | 色值 | 适用场景 |
|:---|:---|:---|
| 纯电 EV | #22a568 | 纯电动汽车 |
| 增程 EREV | #007eba | 增程式电动车 |
| 插混 PHEV | #c172d4 | 插电式混合动力 |
| 燃油 ICE | #ffa200 | 燃油汽车 |

### 4.4 字体规范

## 6. 数据库设计

### 6.1 集合结构

#### cars（车型表）

| 字段 | 类型 | 必填 | 说明 |
|:---|:---:|:---:|:---|
| _id | string | ✅ | 系统唯一标识 |

## 7. 关键业务流程

### 7.1 用户登录流程

```
小程序启动
  ↓
调用 getOpenid 云函数获取 OpenID
  ↓
检查本地缓存是否有用户资料（头像+昵称）
  ↓
是 → 恢复登录状态，进入正常使用
否 → 视为游客，限制功能访问
```

### 7.2 提交评价流程

```
用户点击"撰写评价"
  ↓
检查是否已登录 → 否 → 弹出完善资料弹窗
  ↓
检查是否已评价过此车型 → 是 → 进入编辑模式
  ↓
第一步：五维评分（滑块+快捷按钮）
  ↓
第二步：撰写文字评价
  ↓
提交评价到 reviews 集合
  ↓
调用 updateCarScore 云函数更新车型平均分
  ↓
刷新页面数据
```

### 7.3 添加车型流程

```
用户点击 TabBar + 按钮
  ↓
检查登录状态 → 未登录跳转"我的"页
  ↓
输入品牌 → 显示本地联想列表
  ↓
输入车型 → 基于品牌显示车型联想
  ↓
选择动力形式、填写年款和价格
  ↓
上传车型图片（可选）→ 限10张/人/天
  ↓
提交前调用 suggestCarModels 进行智能纠错
  ↓
跳转评分与评论页
  ↓
通过 submitPendingCarReview 一次性写入 pending 车型 + pending 首评
  ↓
管理员审核通过后对外展示
```

### 7.4 车型图片提交与审核流程

```
用户在车型详情页长按图片
  ↓
选择“提交纠错/新图”
  ↓
图片进入 image_submissions 队列
  ↓
管理员在审核中心逐条查看待审核图片
  ↓
通过 → 覆盖 cars.image_url 成为正式图
拒绝 → 清理待审核图片并保留当前正式图
```

### 7.5 审核中心处理流程

```
管理员进入“我的”页 → 审核中心
  ↓
加载两类任务：
  1. 新车型整包审核
  2. 车型图片审核
  ↓
执行通过 / 拒绝
  ↓
新车型通过后立即写回首评分数和榜单字段
图片通过后替换正式图
```

## 8. 安全与风控

### 8.1 数据安全

| 措施 | 说明 |
|:---|:---|
| 数据库权限控制 | reviews仅创建者可写，cars仅管理员可写 |
| 云函数权限验证 | 敏感操作（删除/修改）通过云函数验证用户身份 |
| 图片上传限制 | 每人每天最多上传10张图片，防止滥用 |
| 头像昵称规范 | 使用微信官方接口获取，避免违规内容 |
| 审核队列隔离 | `image_submissions` 与 `admin_users` 建议关闭客户端读写权限 |

### 8.2 内容审核

| 内容类型 | 审核方式 |
|:---|:---|
| 用户添加车型 + 首评 | 整包进入管理员审核中心，通过后展示 |
| 已存在车型评价 | 保留内容安全预留入口，当前以产品人工巡查兜底 |
| 上传图片 | 进入 `image_submissions` 队列，由管理员逐条审核 |

> 说明：`checkContentSecurity` 云函数已预留接入，但因 `msgSecCheck` 权限问题当前未作为上线阻断条件，现阶段以人工审核兜底。
