# 🚗 车评侦探 - 微信小程序

[![Version](https://img.shields.io/badge/version-v1.4.2-orange)](./)
[![GitHub](https://img.shields.io/badge/GitHub-DHFS/wechattinyapp__review4cars-blue?logo=github)](https://github.com/DHFS/wechattinyapp_review4cars)

一款基于微信原生开发的车型评价小程序，采用 **Editorial Industrialism** 设计风格，让用户可以对不同车型进行五维度评分，查看排行榜，管理个人评价，并通过审核中心管理新车型与车型图片的 UGC 提交。

---

## ✨ 功能特性

### 核心功能
- **车型排行榜** - 基于 `rank_score` 的车型榜单，支持正式榜/观察状态
- **官方图片系统** - 开发者上传车型官方图片，云存储 + 自动转换
- **五维评分系统** - 动力三电(30%)、操控底盘(20%)、空间内饰(20%)、辅驾安全(20%)、其他体验(10%)
- **雷达图展示** - 可视化展示车型在各维度的表现
- **用户评价** - 支持文字评价和五维打分，每位用户每车型限评一次
- **审核中心** - 管理员在小程序内统一审核新车型整包提交和车型图片补充
- **图片审核队列** - 车型图片支持多人多张提交，进入 `image_submissions` 审核队列后逐条审核
- **历史评价档案页** - 档案风格展示、状态标签、分享车型详情
- **个人中心** - 精简“我的”页布局，统一与历史评价页的字重和按钮风格
- **车型录入辅助** - 添加车型时支持品牌/车型本地联想、自动带出动力形式和售价区间、提交前纠错推荐
- **新车型整包审核** - 新车型 + 首次评分 + 首条评论打包提交审核，审核通过后再公开展示


## 🚀 快速开始

### 1. 克隆项目
```bash
git clone https://github.com/DHFS/wechattinyapp_review4cars.git
cd wechattinyapp_review4cars
```

### 2. 导入项目
1. 打开微信开发者工具
2. 选择「导入项目」
3. 选择本项目目录
4. 填写 AppID（需开通云开发）

### 3. 开通云开发
1. 点击开发者工具「云开发」按钮
2. 按指引开通云开发环境

### 4. 创建数据库集合
在云开发控制台 - 数据库中至少创建以下集合：
- `cars` - 车型表
- `reviews` - 评价表
- `admin_users` - 管理员身份表
- `image_submissions` - 车型图片审核队列表

按需保留的历史/辅助集合：
- `upload_logs` - 上传日志（旧流程兼容）

### 5. 配置数据库权限

**reviews 集合**（普通评价仍允许用户直写自己的记录）:
```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

**cars 集合**（敏感修改走云函数）:
```json
{
  "read": true,
  "write": false
}
```

**image_submissions 集合**（审核队列，建议完全关闭客户端权限）:
```json
{
  "read": false,
  "write": false
}
```

**admin_users 集合**（管理员名单，仅云函数读取）:
```json
{
  "read": false,
  "write": false
}
```

### 6. 部署云函数
在微信开发者工具中，右键以下文件夹选择「创建并部署：云端安装依赖」：

```bash
cloudfunctions/getOpenid
cloudfunctions/getCarReviews
cloudfunctions/getMyReviews
cloudfunctions/deleteReview
cloudfunctions/updateReview
cloudfunctions/updateCarScore
cloudfunctions/submitPendingCarReview
cloudfunctions/reviewCarSubmission
cloudfunctions/suggestCarModels
cloudfunctions/updateCarImage
cloudfunctions/checkContentSecurity
```

### 7. 导入车型数据
使用 `cars-batch-import.json` 导入初始车型数据。

## 🎨 设计规范

### 设计风格
**Editorial Industrialism** - 精准档案风格

将小程序从普通的"应用感"转变为高端汽车杂志/技术档案的视觉体验。结合工业机械的粗犷感与高端汽车杂志的精致排版。

### 核心原则
- **无分割线设计** - 仅用色块和留白区分区域
- **大图优先** - 车型封面图作为视觉焦点
- **深色沉浸** - Carbon & Combustion 主题色调
- **编辑式排版** - 杂志级别的文字层级

### 色彩系统

| Token | 色值 | 用途 |
|-------|------|------|
| `surface` | #131313 | 页面背景 |
| `surface-container-low` | #1c1b1b | 卡片背景 |
| `on-surface` | #e5e2e1 | 主文字 |
| `primary` | #ff6b35 | 强调色（评分、按钮） |

**动力类型标签色**:
| 类型 | 色值 |
|------|------|
| 纯电 EV | #22a568 |
| 增程 EREV | #007eba |
| 插混 PHEV | #c172d4 |
| 燃油 ICE | #ffa200 |

### 组件规范

**车型卡片**:
- 背景: #1c1b1b
- 圆角: 32rpx
- 图片高度: 520rpx（占满上半部分）
- 评分数字: 96rpx，#ff6b35
- 无分割线，仅用间距区分

**动力类型标签**:
- 形状: 胶囊形（border-radius: 9999rpx）
- 背景透明度: 0.25
- 边框: 2rpx 实线
- 文字: 白色 + 阴影
- 毛玻璃效果: backdrop-filter: blur(12px)

## 🛠 技术栈

- **框架**: 微信小程序原生开发 (WXML + WXSS + JS)
- **后端**: 微信云开发 (CloudBase)
- **数据库**: MongoDB (云数据库) - `cars`, `reviews`, `admin_users`, `image_submissions`, `upload_logs`
- **存储**: 微信云存储（车型图片、用户头像）
- **云函数**: 11 个（登录、评价管理、分数计算、审核流、图片队列、内容安全预留等）

## ⚠️ 注意事项

### 游客权限边界
为确保数据安全和用户体验，游客（未登录用户）的权限严格受限：

| 功能 | 游客 | 已登录用户 |
|------|------|-----------|
| 浏览排行榜 | ✅ 允许 | ✅ 允许 |
| 查看车型详情 | ✅ 允许 | ✅ 允许 |
| 添加新车型 | ❌ 需登录 | ✅ 允许 |
| 提交评价 | ❌ 需登录 | ✅ 允许 |
| 上传图片 | ❌ 需登录 | ✅ 允许（限10张/天）|
| 查看"我的评价" | ❌ 显示游客模式 | ✅ 显示个人数据 |

**游客访问受限页面时的行为**：
- 点击"添加车型"→弹出登录提示→跳转"我的"页面
- 点击"写评价"→详情页内提示需登录
- 进入"我的"页面→显示游客模式（无数据，有登录按钮）

### 通用注意事项
1. **微信隐私规范**：获取用户信息使用微信官方提供的头像昵称填写能力
2. **数据权限**：数据库 read 权限设为 true（所有人可读），write 权限限制为仅创建者
3. **头像存储**：用户头像上传到云存储，需确保存储权限为"所有用户可读"
4. **图片上传**：返回首页后需要下拉刷新查看新上传的图片
