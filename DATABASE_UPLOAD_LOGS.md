# 上传记录集合 (upload_logs) 说明

## 用途
记录用户上传车型图片的历史，用于实现每日上传次数限制。

## 集合结构

```javascript
{
  _id: "系统自动生成",
  _openid: "用户openid",
  car_id: "车型ID",
  image_url: "图片cloud://路径",
  upload_date: "上传时间（Date类型）",
  created_at: "创建时间（Date类型）"
}
```

## 创建步骤

1. 打开微信开发者工具 → 云开发 → 数据库
2. 点击"添加集合"
3. 输入集合名称：`upload_logs`
4. 点击确定

## 索引建议（可选）

为提高查询效率，建议创建以下索引：

```json
{
  "_openid": 1,
  "upload_date": 1
}
```

## 权限设置

```json
{
  "read": false,
  "write": false
}
```

说明：
- 不允许前端直接读写
- 只能通过云函数操作
- 保护用户上传记录隐私

## 清理策略

上传记录会自动累积，建议定期清理：
- 保留最近30天的记录用于统计
- 或使用云函数定时清理过期数据

## 相关云函数

- `updateCarImage` - 更新车型图片时自动记录
