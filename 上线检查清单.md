## 🔧 上线前必须配置

### 1. 云开发环境
```
微信开发者工具 → 云开发 → 开通
```

### 2. 数据库集合创建
在云开发控制台 → 数据库，创建以下集合：

| 集合名 | 用途 | 权限设置 |
|:---|:---|:---|
| `cars` | 车辆信息 | 所有用户可读，仅创建者可写 |
| `reviews` | 用户评价 | 所有用户可读，仅创建者可写 |

### 3. 权限配置（重要！）

**cars 集合权限：**
```json
{
  "read": true,
  "write": false
}
```

**reviews 集合权限：**
```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

## 📋 审核流程（方案一）

### 用户添加车型后：
1. 用户填写表单 → 点击提交
2. 数据进入 `cars` 集合，`status` 字段为 `pending`
3. 数据**不会**在首页显示

### 管理员审核步骤：
1. 打开 **微信开发者工具**
2. 点击顶部 **"云开发"** 按钮
3. 进入 **数据库** → **cars** 集合
4. 找到 `status` 为 `pending` 的记录
5. 点击 **"编辑"**
6. 将 `status` 从 `pending` 改为 `approved`
7. 点击 **"确定"**
8. 该车型立即在首页显示

## 🚀 提审发布步骤

### 第一步：上传代码
```
微信开发者工具 → 右上角 "上传" 按钮
填写版本号（如 1.0.0）和项目备注
```

### 第二步：微信公众平台配置
1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入 **开发** → **开发管理** → **开发设置**
3. 配置服务器域名（如有外部接口）
4. 配置 **云开发环境 ID**

### 第三步：提交审核
```
微信公众平台 → 版本管理 → 提交审核
```

### 第四步：等待审核通过
- 一般 1-3 个工作日
- 通过后点击 **"发布"**

## ⚠️ 注意事项

### 1. 基础库版本
确保 `app.json` 中要求的最低基础库版本用户能够支持

### 2. 测试账号
审核时可能需要提供测试账号，建议：
- 准备一个简单的测试账号说明文档

### 3. 内容合规
- 确保车型信息真实准确
- 用户评价需要审核机制（已内置）

### 4. 首次数据
建议先手动添加 3-5 条已审核的车型，让首页不空白：
```
在云开发控制台 → cars 集合 → 添加记录
确保 status 字段为 approved
```

## 📞 问题排查

如果首页不显示数据：
1. 检查云开发是否开通
2. 检查 cars 集合是否有 `status: approved` 的数据
3. 检查数据库权限设置

如果无法提交评价：
1. 检查 reviews 集合权限
2. 检查是否获取到用户头像和昵称
